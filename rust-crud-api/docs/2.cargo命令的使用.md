## 命令分解

### `cargo run`
- `cargo` 是 Rust 的包管理器和构建工具
- `run` 是 cargo 的子命令，用于编译并运行 Rust 程序

### `--bin hyperliquid_demo`
- `--bin` 参数指定要运行的二进制程序（binary）
- `hyperliquid_demo` 是二进制程序的名称

## 为什么需要 `--bin` 参数？
从 Cargo.toml 可以看到，你的项目有多个二进制程序：
1. **默认程序**：`rust-crud-api`（主程序，在 `src/main.rs`）
2. **migrate**：数据库迁移工具（在 `src/bin/migrate.rs`）
3. **hyperliquid_demo**：Hyperliquid SDK 演示程序（在 `src/bin/hyperliquid_demo.rs`）

## 不同的运行方式

### 1. 运行默认程序（主 API 服务器）
```bash
cargo run
# 或者
cargo run --bin rust-crud-api
```

### 2. 运行特定的二进制程序
```bash
cargo run --bin hyperliquid_demo    # 运行 Hyperliquid 演示
cargo run --bin migrate             # 运行数据库迁移工具
```
## 总结

`cargo run --bin hyperliquid_demo` 命令的作用是：

1. **编译** Hyperliquid 演示程序
2. **运行** 该程序，展示如何使用 Hyperliquid SDK
3. **测试** SDK 集成是否正常工作

这个演示程序会：
- 连接到 Hyperliquid API
- 获取市场元数据（如 BTC、ETH 等资产信息）
- 获取实时价格数据
- 验证 SDK 功能是否正常

这是一个独立的测试程序，不会启动你的主 API 服务器，只是用来验证 Hyperliquid SDK 的集成效果。
