在 Rust 的 Cargo.toml 文件中，`[package]` 和 `[lib]` 是两个不同的配置节，它们有明确的区别和用途：

## `[package]` 节
- **作用**：定义整个包的基本信息和元数据
- **必需性**：每个 Cargo.toml 都必须有 `[package]` 节
- **内容**：
  - `name`: 包的名称
  - `version`: 版本号
  - `edition`: Rust 版本（如 2015、2018、2021）
  - `authors`: 作者信息
  - `description`: 包描述
  - `license`: 许可证
  - 等等

## `[lib]` 节
- **作用**：专门配置库（library）的构建设置
- **可选性**：只有当包要作为库被其他项目引用时才需要
- **内容**：
  - `name`: 库的名称（可以与包名不同）
  - `path`: 库源文件的路径（默认是 src/lib.rs）
  - `crate-type`: 库的类型（如 "lib", "dylib", "staticlib" 等）

## 在你的项目中

```toml
[package]
name = "axum-crud"        # 包名，用于 crates.io 发布
version = "0.1.0"         # 包版本
edition = "2021"          # 使用 Rust 2021 版本

[lib]
name = "axum_crud"        # 库名（注意下划线）
path = "src/lib.rs"       # 库的入口文件
```

这种配置意味着：
1. 这是一个名为 "axum-crud" 的包
2. 它同时也是一个库，库名为 "axum_crud"
3. 其他项目可以通过 `use axum_crud::...` 来引用这个库的代码
4. 项目结构是"库+二进制"混合模式，既有库代码（src/lib.rs）也有可执行文件（src/main.rs）

这种混合模式的好处是：
- 可以将核心功能放在库中，便于单元测试
- 可以将服务器启动代码放在 main.rs 中
- 其他项目可以直接引用这个库来复用功能
